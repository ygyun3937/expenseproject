# 경비 정산 프로그램 — 환율 산정 방식

작성일: 2026-04-14

## 1. 환율 출처 (변경 예정)

| 구분 | 변경 전(현재) | 변경 후 |
|---|---|---|
| 1차 소스 | Frankfurter API (`api.frankfurter.dev`) | **서울외국환중개** (`smbs.biz`) |
| 2차 소스(Fallback) | Open ER-API (`open.er-api.com`) | **제거** (회사 기준상 단일 공신력 소스만 사용, 장애 시 수동 입력) |
| 호출 방식 | 브라우저(JS) → 외부 API 직접 호출 | 브라우저 → 내부 서버(`/api/exchange-rate`) → 서울외국환중개 스크래핑 |
| 기준 환율 | 해당 API 공시값 | **매매기준율** (서울외국환중개 고시) |

### 변경 사유
- 사내 회계/감사 기준상 **공신력 있는 국내 고시환율**(서울외국환중개 매매기준율) 사용 필요
- 해외 무료 API는 소스·시점이 불명확하여 증빙 근거로 부적합

### 참고 URL
- 서울외국환중개 매매기준율 고시: http://www.smbs.biz/ExRate/StdExRate.jsp

## 2. 환율 적용 로직

### 2-1. 단건 실시간 환율 (`autoFetchRate`)
해외 경비 표의 각 행에서 통화 입력 후 🔄 버튼 클릭 또는 자동 호출 시 동작.

- 입력: 통화코드(예: USD, JPY, EUR)
- 출력: 해당 통화 1단위 = ? KRW (매매기준율)
- 적용 시점: **최근 영업일 고시환율** (주말·공휴일은 직전 영업일)
- KRW 입력 시: 환율 = 1 로 고정

### 2-2. 출장기간 평균 환율 (`fetchAverageRate`)
[평균 환율 일괄 연동] 버튼 클릭 시 동작.

- 입력: 출장 시작일, 도착일, 표에 입력된 외화 통화 목록
- 계산식:

  ```
  평균환율 = Σ(기간 내 각 영업일 매매기준율) / 영업일 수
  ```

- 주말·공휴일은 고시환율이 없으므로 **영업일만** 합산·평균
- 통화별로 계산하여 동일 통화의 모든 행에 일괄 적용

### 2-3. 원화 환산 금액 계산
표에 기입된 외화 금액을 환율과 곱하여 원화 표시.

- 일반 해외경비:
  ```
  원화금액 = 외화금액 × 환율   (round → 원 단위)
  ```
- 해외출장 일비/숙박 (일수 × 단가):
  ```
  원화금액 = 일수 × 외화단가 × 환율   (round → 원 단위)
  ```

### 2-4. JPY(일본 엔) 처리 주의
서울외국환중개는 엔화를 **100 JPY 기준**으로 고시하므로, 1 JPY 환산 시 `고시값 ÷ 100` 변환 필요.

## 3. 시스템 구현 구조 (변경 후)

```
[브라우저 index.html]
    │  통화/기간 전달
    ▼
[Flask 서버 app.py]
    │  /api/exchange-rate              (단건, 최근 영업일)
    │  /api/exchange-rate/average      (기간 평균)
    ▼
[서울외국환중개 smbs.biz HTML 스크래핑]
    │  매매기준율 파싱
    ▼
[서버 응답: { currency, rate, date }]
    ▼
[브라우저: 원화 환산 자동 계산]
```

- 서버에서 호출하는 이유: 브라우저에서 직접 호출 시 **CORS 차단**
- 동일 일자·통화 요청은 서버 메모리 캐시 **TTL 1일** 적용 (매매기준율은 영업일 1회 고시이므로 실시간 재조회 불필요)
- Fallback 미사용 — smbs.biz 응답 실패 시 사용자에게 오류 안내 후 수동 입력

## 4. 증빙·감사 대응

- 각 환율 조회 결과에 **고시일자**와 **출처(서울외국환중개)**를 함께 표시
- 정산서 출력 시 적용 환율·일자가 함께 인쇄되도록 반영
- 기간 평균 사용 시: 사용한 영업일 수와 평균값 근거 명시 가능
