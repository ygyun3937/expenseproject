# 영수증 경비정산서 자동화 시스템 설계

**날짜:** 2026-04-11  
**상태:** 확정

---

## 1. 개요

회사 직원(60명)이 영수증 이미지를 업로드하면 Gemini AI가 자동으로 항목을 인식하고, 검토 후 회사 양식에 맞게 경비정산서를 인쇄할 수 있는 로컬 웹 앱.

---

## 2. 사용자 워크플로우

```
영수증 이미지 업로드 (여러 장 동시)
        ↓
이미지 전처리 (자동, 백엔드)
        ↓
Gemini 2.0 Flash API 호출 (자동, 백엔드)
        ↓
결과 테이블 화면 표시 (가게명, 날짜, 항목, 금액)
        ↓
사용자 검토 및 수정 (인라인 편집)
        ↓
[인쇄] 버튼 클릭 → 브라우저 인쇄 페이지 → 회사 양식 PDF 출력
```

---

## 3. 아키텍처

### 플랫폼
- **런타임**: Python 3.10+ 로컬 서버 (Flask)
- **프론트엔드**: HTML/CSS/Vanilla JS (설치 불필요)
- **배포**: PyInstaller로 `.exe` 단일 파일 패키징 (Python 설치 불필요, ~50MB)
- **접속**: 실행 후 자동으로 브라우저 오픈 (`localhost:5000`)

### 기술 스택
| 역할 | 라이브러리 |
|------|-----------|
| 웹 서버 | Flask |
| 이미지 전처리 | Pillow |
| AI 인식 | google-generativeai (Gemini 2.0 Flash) |
| 패키징 | PyInstaller |
| 인쇄/PDF | 브라우저 기본 인쇄 기능 (CSS `@media print`) |

## 5. 영수증 인식 전략

### 5-1. 이미지 전처리 (Pillow)
- 해상도 표준화: 장변 기준 최대 2048px로 리사이즈
- 밝기/대비 자동 보정 (`ImageEnhance`)
- EXIF 회전 정보 반영 자동 보정

### 5-2. 구조화 프롬프트 (코드에 고정, 사용자 개입 없음)
```
한국어 영수증 이미지입니다.
다음 항목을 JSON 형식으로 정확하게 추출해 주세요:
{
  "store_name": "가게명",
  "date": "YYYY-MM-DD",
  "items": [
    {"name": "품목명", "quantity": 수량, "unit_price": 단가, "amount": 금액}
  ],
  "subtotal": 소계,
  "tax": 부가세,
  "total": 합계
}
인식할 수 없는 항목은 null로 표시해 주세요.
반드시 JSON만 반환하고 다른 설명은 포함하지 마세요.
```

### 5-3. 결과 처리
- JSON 파싱 후 화면 테이블에 표시
- 파싱 실패 시 해당 셀을 빈칸으로 표시 (사용자 수동 입력 유도)
- 인식 오류는 사용자가 인라인 편집으로 수정

---

## 6. 인쇄 / PDF 출력

- 브라우저 기본 인쇄 기능 활용 (`window.print()`)
- CSS `@media print` 로 회사 양식에 맞는 레이아웃 적용
- 추후 회사 양식 제공 시 인쇄 CSS 수정으로 유연하게 대응

## 8. 설정 파일 (`.env`)

`.exe` 옆에 `.env` 파일을 함께 배포. 코드 수정/재빌드 없이 텍스트 파일만 수정하면 됨.

```
GEMINI_API_KEY=your-api-key-here
GEMINI_MODEL=gemini-2.0-flash   # 모델 업데이트 시 이 줄만 변경
```

- 모델 변경: `GEMINI_MODEL` 값만 수정
- API 키 변경: `GEMINI_API_KEY` 값만 수정
- 관리자만 `.env`를 관리하면 전직원 일괄 적용 가능 (중앙 서버 배포 시)

---

## 9. 배포

1. PyInstaller로 `.exe` 빌드
2. `.exe` + `.env` 파일 세트를 직원에게 공유 (USB, 사내망, 이메일 등)
3. 더블클릭 → 브라우저 자동 오픈 → 사용

---

## 10. 확장 가능성

- 현재: 개인 PC 로컬 실행
- 추후: 중앙 서버에 배포 시 코드 변경 없이 그대로 사용 가능 (Flask → Gunicorn/nginx)
- 양식 변경: 인쇄 CSS + 프롬프트 수정만으로 대응
