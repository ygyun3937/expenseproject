## [슬라이드 4] 1. 환율 자동 조회 — 서울외국환중개 매매기준율

### Before
- Frankfurter / open.er-api 등 해외 무료 API 사용
- 출처 불명확 → 감사·회계 증빙 시 설명 어려움

### After
- **서울외국환중개(`smbs.biz`) 매매기준율** 단일 소스
- **출장 기간 평균 환율 일괄 연동** 버튼
- **JPY 자동 환산** (고시 100엔 → 1엔 기준)
- 1일 캐시로 서울외국환중개 사이트 부하 최소화
- 장애 시 수동 입력 fallback

### 효과
- 감사 시 "어디 값이냐?" 질문에 **공식 기관 고시**로 답변 가능
- 기간 평균 자동 계산으로 출장 경비 정확도 향상

## [슬라이드 5] 2. 식비 한도 자동 반영 — 1인당 12,000원

### 로직
- `식비 금액 > 한도 × 인원수` 이면 **한도까지만** 셀에 반영
- 비고에 `[한도적용 원금 15,000]` 자동 기록 (감사 추적용)
- AI 업로드 / 수동 입력 / 저장 복원 모두 적용

### 예시
| 상황 | 기록 결과 |
|---|---|
| 1인 · 15,000원 | 식비 **12,000원** + 비고 「한도적용 원금 15,000」 |
| 3인 · 40,000원 | 식비 **36,000원** + 비고 「한도적용 원금 40,000」 |
| 3인 · 30,000원 | 식비 **30,000원** (한도 이내, 그대로) |

### 관리자 설정
- 한도는 `.env` 파일의 `MEAL_LIMIT_PER_PERSON` 으로 중앙 관리
- 회사 규정 변경 시 서버 관리자가 1줄 수정

## [슬라이드 6] 3. 영수증 AI 인원수 인식

- 영수증에 **"3명" "3인" "3인분" "인원: 3"** 등 표기 시 Gemini가 자동 인식
- **파일명**에 `점심_3명.jpg` 형식도 인식
- 메모에 `[3명]` 자동 부착 → 1인당 정산 계산 근거

### 운영 팁
- 영수증에 손글씨로 인원수 기재 권장
- 파일명 표기도 병행 시 인식률 ↑

## [슬라이드 7] 4~6. 사용성 개선

### 주차비 매핑 버그 수정
- 기존: AI가 주차비 인식 시 교통비 칼럼에 들어감 (인덱스 오류)
- 수정: 헤더 순서와 정확히 일치 (교통비=3, 주차비=4)

### 영수증 추가 업로드 시 기존 데이터 유지
- 기존: 추가 업로드 시 모든 행이 초기화됨 → 사용자 반복 작업
- 수정: 기존 행 유지 + 새 영수증만 추가, 중복 자동 탐지

### 임시저장 전체 범위 확장
- 기존: 개인경비 표만 저장
- 수정: 작성자·차량·유류비 로그·출장일비·해외 경비 모두 저장

## [슬라이드 8] 7. 유류비 정산 반영 자동화

### 유류비 → 개인경비 정산표 자동 이전
- 주행 로그 · 통행료 합산 → **정산 반영** 버튼 한 번에 개인경비 표로 이관

### 날짜 자동 입력 (회계 정산 차수 기준)
- **1차 정산 (1~15일)** → 작성일 해당 월 **15일**
- **2차 정산 (16~말일)** → 작성일자 **전달(이전 달) 말일**

### 효과
- 정산 차수에 맞는 날짜 기입 실수 제거
- 회계팀 검증 편의성 향상

## [슬라이드 9] 8. 인쇄·PDF 출력 최적화

### 자동 최적화 7가지
1. 정산 금액 **0원인 탭 자동 제외** (국내 또는 해외 단독 출력)
2. 유류비 내역 없으면 **유류비 페이지 생략**
3. 출장일비 금액 0원 시 **출장일비 섹션 제외**
4. 비어있는 입력 행 **인쇄 숨김**
5. 식비 한도 초과 빨간 경고 배경 **인쇄 시 중립화**
6. 비고 칼럼 **40% 우선 배정** + 나머지 균등 분배
7. 합계 행 배경색을 테이블 제목색과 **통일**

### 결과
- 불필요한 빈 페이지 제거
- 깔끔한 증빙 출력물 (회계 제출 그대로 사용 가능)
